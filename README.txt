Тезис
------
Основной объект системы - документ, содержащий тезис доклада который
юзер собирается делать на конференции. Документ(далее Тезис) посылается юзером на утверждение экспертной
группе. Эксперты могут обсуждать Тезис в рамках системы и затем принять
его или отклонить.
Тезис имеет основные атрибуты: дата создания, автор, название, описание, приаттаченный файл.   
Тезис должен быть в одном из статусов: обсуждается, отклонён,принят.
Работа со статусом должна быть по правилам:
1) При создании Тезиса он сразу имеет статус "обсуждается"
2) При каждом назначении статуса в системе также фиксируется: кто-когда,опциональное примечание .
3) При смене статуса все характеристики прежнего статуса заносятся в историю статусов.
4) Статусы "отклонён" и "принят" могу назначать только эксперты


Комментарий к тезису
---------------------
Тезис можно комментировать(это могут делать только
эксперты).
Комментарий имеет основные атрибуты: кто и когда, текст комментария


Интерфейс пользователя
----------------------
Доступ: пользователь
При входе в личный кабинет (ЛК) открывается страница со списком
тезисов пользователя + кнопка "добавить тезис". 
В действительности у большинства будет не более одного тезиса.
По тезису отображается: 
1) ID тезиса, Наименование, Описание, Файл, 
2) Текущий статус (со всеми его параметрами) + История статусов



Интерфейс эксперта
------------------
Доступ: эксперт
Такой же как у обычного пользователя с дополнительным меню "Тезисы" с пунктами
1) на обсуждении
2) принятые
3) отклонённые
каждый содержит список тезисов в соответствующих статусах (педжинация
естественно нужна).
основные колонки: ID тезиса, Дата-Время, Наименование, Автор, Кол-во комментариев.
при клике на Наименование открывается страница Обсуждения тезиса.


Страница Обсуждения тезиса
--------------------------
Доступна: эксперт
В начале страницы приводиться информация о тезисе:ID, название +
описание, ссылка на файл, кто-когда добавил, история статусов(включая
текущий).
Для экспертов отображается также две ссылки - "отклонить" и "принять" которые ведут на простые страницы-формы с полем "примечание".
Далее идут комментарии.
Далее идёт форма для отправки комментария.


Отсыл email-сообщений
----------------------
email-сообщения отсылаются в следующих случаях:

1) При смене статуса тезиса сообщение об этом отсылается его автору, при
условии что статус сменил не он. 
текстовка: "вашему тезису N ID-тезиса присвоен статус: статус. пояснение эксперта: комментарий если есть"
текстовка должна быть переводимой.

2) Когда эксперт принимает\отклоняет тезис, сообщение рассылается всем
остальным экспертам
текстовка: "тезису N ID-тезису(ссылка) присвоен статус: статус. автор: эксперт такой-то. пояснение эксперта: комментарий если есть".

3) При комментировании тезиса сообщение об этом отсылается всем
остальным экспертам. 
текстовка: "для тезиса N ID-тезиса (ссылка) был добавлен комментарий.автор: эксперт такой-то"
